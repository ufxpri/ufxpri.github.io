Redis vs Memcached
redis - 복제되는 캐시
- multiAZ Auto-Failover
- 읽기 복제
- AOF persistence 로 데이터 안정성
- 백업 복구 지원

memcached - 샤딩으로 연결
- HA 없음
- 영속성없음
- 백업 없음
- 멀티 스레딩임


솔루션 설계자를 위한 Elasticache

캐시 보안
- redis 를 위한 iAM 인증을 지원한다.
- IAM 정책은 API-level 보안에서만 사용된다.
- RedisAUTH
	- redis 클러스터를 만들때 id/pw 를 설정할 수 있다.
	- 추가적인 레벨의 보안 수준이다.
	- SSL 을 지원한다.
- memcached
	- SASL-기반의 인증을 지원한다.

캐시 사용 패턴
1. 지연 로딩: 모든 데이터는 캐시로 읽어진다.
2. write through: db에 쓰기전 캐시에 쓴다.
3. 세션 저장: TTL과 같은 기능을 사용하여 세션데이터를 저장한다.
