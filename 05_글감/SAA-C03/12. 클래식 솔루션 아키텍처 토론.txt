사례1 whatis the time.com
시간을 알려주는 서비스
데이터베이스 없어도 됨
다운타임을 제거할수록 좋음
서비스가 확장되고 문제가 발생함에 따라 이를 대응하는 방법을 제안하는게 우리가 할 일이다.


사례2 mycloths.com
상태 웹사이트
확장가능해야하고 HA를 달성해야함
stickiness 를 사용하여 사용자와 인스턴스를 고정
이 대신 장바구니를 사용자 측에 저장하는 방법도 있음
하지만 요청이 계속 무거워짐. 공격도 있을 수 있음
세션관리를 사용하는 방법이 있다 elastic cache 를 사용하여 세션을 저장해두고 사용한다.


사례3 mywordpress.com
이미지 저장 ebs 에 연결되어 있는 사례를 생각해보자
인스턴스에서 바로 ebs에 연결되기 때문에 데이터를 쉽게 가져올 수 있다.
하지만 ALB 를 적용 이후에는 접근할때마다 데이터가 다를 수 있음
이를 여러 인스턴스에서 접근할 수 있도록 efs를 사용하여 접근할 수 있도록 함
aurora 데이터베이스가 접근이 쉽다.


어플리케이션을 인스턴스화 하기
어플리케이션을 서비스화 하는것은 많은 작업을 필요로 한다.
- golden AMI : 인스턴스를 런칭하기 전에 다 설치하고 AMI 로 배포
- bootstrap using user data: 사용자 데이터 스크립트를 사용하여 설정
- hybrid: golden ami 와 user data (Elastic Beanstalk)
