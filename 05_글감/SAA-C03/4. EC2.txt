aws 는 퍼블릭IP 와 프라이빗IP 를 가지고 있다.

aws 안에서는 private IP 로 통신할 수 있다.

모든 컴퓨터는 private network 안에 있기 때문에 서로 통신하지만

gatway 를 거치면 외부로 나갈 수 있다.

aws 도 동일하다. InternetGateway 를 거쳐 외부로 나갈 수 있다.

public ip
- 외부에 공개됨
- 인터넷 상에서 특수함

private ip
- 내부에서만 하용됨
- 외부 접근 불가능

elastic ip
- 5개까지만 사용 가능
- dns 를 권장함
- load balancer 도 권장함

placement group이란?
EC2 가 배치되는 전략을 설정하고 싶을때
- cluster - 하나의 AZ 인 저지연 그룹에 할당
	같은 랙 같은 서버. 저지연 통신이 보장된다.
	하드웨어가 망가지면 다 망가진다.
	그래서 빠른 네트워크가 필요한 연산작업에 사용된다.
- spread - AZ당 최대 7개 의 인스턴스에 공고르 배치
	완전 반대. 실패를 최소화 한다. 모든 인스턴스가 다른 AZ에 할당된다.
	다른 AZ에서도 각각 다른 랙에 할당된다.
	AZ당 7개인스턴스로 제한된다.
	HA 를 강화할떄 사용한다.
- partition - 여러 가용영역에 분산가능
	여러 AZ에 partition을 만들고 instance 를 만들 수 있다.
	partition 은 랙을 의미한다.
	그래서 파티션간은 분리되어있어 서로 안전할 수 있다.
	100개의 인스턴스까지 지원한다.
	빅데이터 같은 경우 사용한다.
	파티션 내부는 같은 랙에 있기 떄문에 속도가 매우 빠르다.
	


Elastic Network Interface (ENI)
가상 네트워크 카드 같은 것이다.

Eth0 에 기본적으로 연결되어 IP를 할당받아 사용하고 있을 것 이다.
주 IPv4 주소와 보조 IPv4 주소를 가질 수 있다.

개별로 생성하여 할당할 수 있다.

보조 IP 는 손으로 옮길 수 있기 떄문에 고정 ip 로 서비스 하더라도 장애조치가 가능하다. ENI 만 옮기면 되니까!


EC2 Hibernate
인스턴스를 종료하지 않고 나중에 다시 시작할 수 있도록 하고싶다!

기존 방식대로 종료 및 재시작 하면 초기화 과정을 다 거친다.

hibernate 는 메모리 상태까지 저장하기때문에 빠른 재시작이 가능하다.

이는 ram 의 내용이 암호화되어 root EBS 에 저장되기 떄문에 가능하다.

따라서 EBS 는 반드시 암호화가 설정되어 있어야 한다.
