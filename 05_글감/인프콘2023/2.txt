API first design 정의하기! 계약서에 대한 이해가 필요하다. 계약서에는 이해 관계자 관이 합의하에 계약 조건을 논의하고 작성한다. API 문서 또한 위배되서는 안되는 약속이다. 따라서 이를 API 계약서 ㅏㄹ고 부르겠다. 이 문서는 계약서와 같은 효력을 발휘하여 이 API 가 유효하기 위해서 필요하게 저해진 기준이 있고 그에따라 API 를 작성할 수 있다. 이 기준을 OpenAPI Specification (OAS)  라고 한다. 언어에 구애받지 않는 http API 표준 인터페이스 이다 를 나타낸다. api first design 이란 무엇이지? API 계약서를 1순위로 하고 개발하는것을 의미한다. 이를 활용해서 문제를 해ㄹ하는 방법을 알아보자. 개발 프로세스틑 다음과 같다 백과 포런트가 요구사항을 기반으로 요구사항 정의서를 작성한다. 이를 공유하고 토론하ㅕ API 계약서를 다듬자. 어느정도 정해지면 openAPOI  도구를 사용하여 작성한다.
