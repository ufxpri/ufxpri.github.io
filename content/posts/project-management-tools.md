# 프로젝트 관리 도구

프로젝트 관리를 위한 다양한 도구들을 비교하고 각각의 특징을 정리했습니다.

## [Github](https://github.com/)
- 레포지토리 지원
- 프로젝트 칸반 지원
- 이슈 지원
- 마일스톤
- [git 배우기](https://learngitbranching.js.org/)
- [issue 관리 가이드](https://www.popit.kr/github로-프로젝트-관리하기-part1-이슈-발급-부터-코드리뷰까/)

## [ZenHub](https://www.zenhub.com/)
- github 확장 버전
- 이슈 시각화
- 프로젝트 진행 상황을 더 직관적으로 확인 가능

## [Freedcamp](https://freedcamp.com/)
다양한 기능을 제공하는 올인원 프로젝트 관리 도구:
- 할일 목록
- kanban 보드
- 단계적 작업
- 프로젝트 차트
- 개인 작업 목록
- 캘린더
- 토의
- 자체 위키
- 벽 포스트 (대시보드)
- 이슈 트래커
- 시간 추적
- 마일스톤

## [Trello](https://trello.com/)
- 칸반 도구의 대표주자
- 무료
- 심플하고 직관적인 인터페이스
- 다양한 고급 기능은 없지만 칸반 보드로는 충분

## [Asana](https://asana.com/)
- 칸반 도구
- 태스크 관리에 특화
- 팀 협업 기능 강화

## [Slack](https://slack.com/)
- 채팅 기반 커뮤니케이션 앱
- 많은 확장성 (텍스트형식 봇)
- 다양한 서비스와 통합 가능
- 실시간 소통에 최적화

## [Jira](https://ko.atlassian.com/software/jira)
- 애자일 보드 사용 가능
- 로드맵 지원
- 강력한 이슈 트래킹
- 엔터프라이즈급 기능
- 많은 수의 번들을 구매해야 하는 단점

## [Notion](https://www.notion.so/)
올인원 워크스페이스:
- slack 형 문서관리
- markdown 지원
- board 지원
- 일정 공유
- 테이블/데이터베이스 기능
- 파일 공유
- 위키, 문서, 칸반 등 다양한 형태로 활용 가능
- 처음부터 설계해야 하는 불편함이 있음
- 디자인이 한눈에 안 들어올 수 있음

# 정리

현재까지 알아본 내용을 바탕으로, 우리 회사는 Github 유료버전을 사용하고 있고 Github를 적극 활용하는게 좋을것으로 보인다. 따라서 **Slack**과 **Github**를 주력 도구로 사용한다.

Notion을 도입하면 조금 더 예쁘게 문서를 관리할 수도 있을것 같지만 일단 제쳐둔다.
