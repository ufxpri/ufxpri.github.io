블로그를 시작하게 된 이유

결론

블로그를 통해서 나와 약속을 하고 싶었다.

기술 블로그

최신 트렌드에 관하여 보고한다. 함수형 프로그래밍 이라던가, 새로운 언어, 기법등을 소개한다.

+alpha

다른사람의 글을 보고 더 배운점을 덧붙혀 볼 수 있는 새로운 프레임을 만들 것이다. 이 목적은 다른사람이 똑같은 튜토리얼을 진행했을때 내가 겪었던 문제나 추가조사로 알게된 정보들을 공유할 수 있도록 하는 것이다.

튜토리얼

내가 보고 실행했던 모든 튜토리얼들을 복제할 생각이다. 어쩌면 복붙처럼 들릴 수 있겠지만 정확히는 튜토리얼을 모두 실행 후 재구성 하는 것이다. 최종 목적은 나 자신이 튜토리얼을 완료하고 잘 기억하고 있는지 확인하는 것이다.

사내 협업 관리 툴 개선방안

현재까지 회사에선 개발 프로세스 없이 프로젝트를 관리하고 소통하고 있었다. 이를 개선할 개선방안을 준비해야겠다고 생각했다.

현재까지의 작업 흐름

프로젝트 사이클

  • 외부에서 요청이 들어옴
  • 담당자 선정
  • 작업
  • 배포

사내에 프로젝트가 하나씩 생길때 마다 그 프로젝트의 담당자를 정하고 시간을 준 뒤 결과를 확인하는 방식이였다. 물론 현재까지의 프로젝트에서 다양한 기능이 포함되거나 복잡한 구조가 필요한 프로젝트는 없었지만 이 때문에 다른사람이 업무를 분담할 수 없게 되었고 그 결과 업무를 맡은 사람이 완성하지 못할 시 그 사람이 업무를 끝낼때 까지 아무도 도움을 주지 못한다는 것이다.

프로젝트 관리 도구

프로젝트 관리를 위한 다양한 도구들을 비교하고 각각의 특징을 정리했습니다.

Github

  • 레포지토리 지원
  • 프로젝트 칸반 지원
  • 이슈 지원
  • 마일스톤
  • git 배우기
  • issue 관리 가이드

ZenHub

  • github 확장 버전
  • 이슈 시각화
  • 프로젝트 진행 상황을 더 직관적으로 확인 가능

Freedcamp

다양한 기능을 제공하는 올인원 프로젝트 관리 도구:

  • 할일 목록
  • kanban 보드
  • 단계적 작업
  • 프로젝트 차트
  • 개인 작업 목록
  • 캘린더
  • 토의
  • 자체 위키
  • 벽 포스트 (대시보드)
  • 이슈 트래커
  • 시간 추적
  • 마일스톤

Trello

  • 칸반 도구의 대표주자
  • 무료
  • 심플하고 직관적인 인터페이스
  • 다양한 고급 기능은 없지만 칸반 보드로는 충분

Asana

  • 칸반 도구
  • 태스크 관리에 특화
  • 팀 협업 기능 강화

Slack

  • 채팅 기반 커뮤니케이션 앱
  • 많은 확장성 (텍스트형식 봇)
  • 다양한 서비스와 통합 가능
  • 실시간 소통에 최적화

Jira

  • 애자일 보드 사용 가능
  • 로드맵 지원
  • 강력한 이슈 트래킹
  • 엔터프라이즈급 기능
  • 많은 수의 번들을 구매해야 하는 단점
